Core Engine

CEMOptimizer

Cross-Entropy Method optimizer. Samples N action sequences, evaluates each against the physics + residual model, keeps the top k elites, and refits a Gaussian for the next iteration.

cfg = PhysiCoreConfig(
    cem_samples=512,    # candidate action sequences per iteration
    cem_elites=64,      # top-k kept for Gaussian refitting
    cem_iterations=5,   # optimization iterations per step
    horizon=20,         # planning horizon (steps)
)

PhysicsLayer

Wraps the platform dynamics function. Applies noise injection for robustness and caches the nominal trajectory for residual computation.

ResidualEnsemble

An ensemble of lightweight linear models that learn the sim-to-real gap online. The ensemble mean is added to the physics prediction; the variance is used as an uncertainty estimate.

OnlineSystemID

Recursive least-squares estimator for physical parameters (mass, drag coefficient, friction). Updates after every real transition.

State Estimator (EKF)

Extended Kalman Filter that fuses observations from multiple PerceptionSource adapters.

from physicore.perception.interface import PerceptionFusion, PoseSource
from physicore.core.state_estimator import StateEstimator

estimator = StateEstimator(state_dim=12, obs_dim=6)
fusion = PerceptionFusion(estimator)
fusion.register(PoseSource())
fusion.start_polling(hz=50)
fused = fusion.fuse()

Fleet Manager

Manages a pool of PhysiCore engine instances, one per robot. Provides unified health monitoring, step dispatch, and cross-robot coordination hooks.

Planning Layer

  • TrajectoryPlanner — joint-space, task-space (via IK), waypoints, circular arcs
  • IKSolver — Jacobian pseudoinverse with null-space joint-limit avoidance
  • TrajectoryExecutor — sync/async execution with abort support
  • ObstacleMap — sphere and box obstacles with path-clearance checks

Plugin System

Plugins are Python packages loaded at runtime. Before installation, every plugin is scanned with an AST walker + regex pass that blocks socket, subprocess, eval, exec, and network libraries. Allowed: open('/plugins/...') for local data access.

Multi-tenant Platform

  • OrgStore — SQLite-backed organizations, memberships, invites, quotas
  • UsageMetering — daily step counters, storage, active robots
  • AuditLog — append-only event log with org/user/action indexes
