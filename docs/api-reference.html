API Reference

The PhysiCore FastAPI server exposes a REST API on http://localhost:8000. Interactive docs available at /docs (Swagger) and /redoc.

Start the server with physicore serve or uvicorn physicore.api.server:app

Core

GET /health

Returns server health and version info.

{ "status": "ok", "version": "1.3.0", "platform_count": 12 }

GET /api/platforms

List all supported platform identifiers.

["balancing_bot", "quadrotor", "fixed_wing", "evtol", "manipulator_arm",
 "legged_robot", "ground_rover", "rocket", "auv", "satellite"]

Fleet

POST /api/fleet/register

Register a new robot in the fleet.

// Request body
{
  "robot_id": "bot-001",
  "platform": "balancing_bot",
  "params": { "mass": 1.0, "friction": 0.15, "inertia": 0.01 }
}
// Response
{ "robot_id": "bot-001", "status": "registered" }

GET /api/fleet/health

Returns health status for all registered robots.

[
  { "robot_id": "bot-001", "platform": "balancing_bot", "status": "healthy", "residual_norm": 0.0012 }
]

POST /api/fleet/{robot_id}/step

Run one MPC step for a specific robot.

// Request body
{ "state": [0.1, 0.0, 0.0, 0.0], "action_hint": [0.0] }
// Response
{ "action": [0.23], "residual_norm": 0.0034, "step_time_ms": 4.2 }

Telemetry

GET /api/telemetry/sessions

List all recorded telemetry sessions.

GET /api/telemetry/sessions/{session_id}

Get all steps for a session.

GET /api/telemetry/sessions/{session_id}/export

Export session data. Query param: format=csv or format=json.

DELETE /api/telemetry/sessions/{session_id}

Delete a session and all its steps.

Trajectory Planning

POST /api/plan/joint

// Request
{ "robot_id": "arm-001", "q_start": [0,0,0,0], "q_goal": [1,0.5,-0.3,0], "duration": 3.0 }
// Response
{ "trajectory_id": "traj-abc123", "n_points": 150, "duration": 3.0,
  "collision_clear": true }

POST /api/plan/task

Plan a trajectory to a task-space target (uses IK internally).

POST /api/plan/waypoints

Plan through a sequence of joint configurations with quintic blending.

POST /api/plan/circular

Plan a circular arc trajectory in task space.

POST /api/execute/{trajectory_id}

Execute a previously planned trajectory. Query param: async_mode=true.

Organizations

POST /api/orgs

Create a new organization. Header: X-User-Id.

GET /api/orgs/{org_id}

Get organization details.

GET /api/orgs/{org_id}/members

List all members and their roles.

GET /api/orgs/{org_id}/usage

Get current usage vs. quota for the organization.

GET /api/orgs/{org_id}/audit

Query the audit log. Query params: action, since, limit.

Marketplace

GET /api/marketplace

Search/list plugins. Query params: q, category, free_only, verified_only.

GET /api/marketplace/{plugin_id}

Get full plugin details including reviews and screenshots.

POST /api/marketplace/{plugin_id}/install

Install a plugin for a user. Header: X-User-Id.

POST /api/marketplace/{plugin_id}/rate

Submit a rating and review for a plugin.

WebSocket

ws://localhost:8000/ws/telemetry

Real-time telemetry stream. The server pushes a JSON frame every control step:

{
  "robot_id": "bot-001",
  "t": 1.24,
  "state": [0.02, -0.01, 0.1, 0.03],
  "action": [0.14],
  "residual_norm": 0.0008
}
Interactive docs: Visit http://localhost:8000/docs while the server is running for a live Swagger UI with try-it-out for every endpoint.
