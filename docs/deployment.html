Deployment

PhysiCore can be deployed as a local process, a Docker container, or a cloud-native service.

Local (Development)

# Start API server
physicore serve --host 0.0.0.0 --port 8000

# Start React dev server (separate terminal)
npm run dev

# Open dashboard
open http://localhost:5173

Docker (Single Container)

# Build
docker build -t physicore:latest .

# Run — serves API + built frontend on :8000
docker run -p 8000:8000 \
  -v ~/.physicore:/root/.physicore \
  physicore:latest

Docker Compose (Recommended)

docker compose up          # foreground
docker compose up -d       # background
docker compose logs -f     # follow logs
docker compose down        # stop
# docker-compose.yml (already included in the repo)
services:
  physicore:
    build: .
    ports: ["8000:8000"]
    volumes:
      - ~/.physicore:/root/.physicore
    environment:
      - PHYSICORE_HOST=0.0.0.0
      - PHYSICORE_PORT=8000
    restart: unless-stopped

Production Hardening

Multiple Workers

physicore serve --workers 4
# or
uvicorn physicore.api.server:app --workers 4 --host 0.0.0.0
Note: The in-memory fleet state is not shared across workers. Use a single worker for fleet management, or move fleet state to Redis/PostgreSQL for multi-worker deployments.

Reverse Proxy (nginx)

server {
    listen 443 ssl;
    server_name api.yourrobot.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 300s;
    }
}

Systemd Service

[Unit]
Description=PhysiCore API Server
After=network.target

[Service]
User=physicore
WorkingDirectory=/opt/physicore
ExecStart=physicore serve --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5
Environment=PHYSICORE_LOG_LEVEL=info

[Install]
WantedBy=multi-user.target
sudo systemctl enable physicore
sudo systemctl start physicore

CI/CD — GitHub Actions

The repo ships two workflows:

  • ci.yml — runs on every push/PR: pytest, TypeScript check, Docker build
  • release.yml — triggered on v* tags: test → Docker push → PyPI publish → GitHub Release
# Tag and release v1.0.0
git tag v1.0.0
git push --tags
# GitHub Actions picks up the tag and runs release.yml automatically
