Safety

PhysiCore is designed for production robotics where failures can damage hardware or injure people. This page covers every safety layer built into the engine.

Warning: PhysiCore is research software. Always validate in simulation before deploying to real hardware. Never disable safety layers in production.

SentinelOS

The runtime safety watchdog. Runs in a background thread and monitors every control step.

from physicore.core.sentinel import SentinelOS, SentinelConfig

sentinel = SentinelOS(SentinelConfig(
    max_residual=0.5,       # trigger if residual_norm exceeds this
    max_action_norm=10.0,   # clip action if norm exceeds this
    watchdog_timeout_s=0.1, # e-stop if no step within 100 ms
    state_bounds={          # per-dimension hard limits
        0: (-1.57, 1.57),   # joint 0: ±90°
        2: (-5.0, 5.0),     # position: ±5 m
    },
))
sentinel.start()

# In control loop:
action = sentinel.check_and_clip(state, action, residual_norm)

State Bounds

Hard limits on individual state dimensions. The sentinel clips actions to prevent bound violations:

cfg = SentinelConfig(
    state_bounds={
        0: (-3.14, 3.14),    # angle: ±π
        2: (-10.0, 10.0),    # position: ±10 m
        1: (-5.0, 5.0),      # angular velocity
    }
)

Action Clipping

All actions are clipped to the platform's defined action space before being sent to the actuators. The CEM optimizer respects these bounds during sampling.

cfg = PhysiCoreConfig(
    action_min=np.array([-10.0]),   # per-dimension lower bounds
    action_max=np.array([10.0]),    # per-dimension upper bounds
)

Residual Watchdog

If the residual norm (sim-to-real error) exceeds a threshold, the sentinel can trigger a configurable response:

  • WARN — log and continue (default below threshold)
  • CLIP — scale action down proportionally
  • ESTOP — zero all actuators immediately
cfg = SentinelConfig(
    residual_warn=0.2,
    residual_estop=0.8,
    on_estop="zero_action",  # "zero_action" | "hold_last" | "raise"
)

Audit Logging

Every org-level action is written to an append-only SQLite audit log:

from physicore.api.audit import get_audit_log

log = get_audit_log()
events = log.query(org_id="org-abc", action="plugin.install", limit=50)
for e in events:
    print(f"{e.timestamp}  {e.user_id}  {e.action}  {e.status}")

Hardware E-Stop Integration

For real hardware, wire your physical e-stop to call sentinel.trigger_estop():

import signal

def on_sigterm(sig, frame):
    sentinel.trigger_estop(reason="SIGTERM received")
    sys.exit(0)

signal.signal(signal.SIGTERM, on_sigterm)
signal.signal(signal.SIGINT, on_sigterm)
Best practice: Always test your e-stop path before a real-robot deployment. Confirm that trigger_estop() reaches the actuators within your hardware's safe response window.

Checklist Before Real Deployment

  • Run 1000+ simulation steps at 10× speed and verify no sentinel triggers
  • Confirm state bounds match your physical workspace
  • Set residual_estop to a value calibrated from sim-to-real gap measurements
  • Test hardware e-stop path end-to-end
  • Enable audit logging for all robot actions
  • Set watchdog_timeout_s below your actuator's safe coastdown time
